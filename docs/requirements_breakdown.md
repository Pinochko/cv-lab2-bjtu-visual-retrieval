# 实验 2 需求拆解

## 已确认要求

- 提交时间：2026-06-16。
- 任务 1：图像检索。
- 任务 2：文字检测/目标检测可视化。
- 报告通过智慧课程平台提交，建议 PDF。
- 报告中不要贴代码。
- 报告中给出开源项目链接。
- 项目中包含源代码、运行说明文档、演示视频。

## 图像检索任务

- 使用 `query` 文件夹中的 135 张图片作为查询。
- 对 `base` 文件夹中的图片进行检索。
- 训练或建库时不能使用类别信息。
- 类别信息只在计算准确率时使用。
- 标签由文件名前缀获得，例如 `fhy-xxx.jpg` 的标签是 `fhy`。
- 评价指标是 Precision@TopK。
- 需要统计 `K = 20, 40, 60`。
- 每类 landmark 生成一个 P@K 图，共 12 张图。

## 文字检测任务

- 使用 `object_detection/data` 中的图片和 LabelMe json 标注。
- 对文字区域进行检测或可视化。
- PPT 中评价方式是可视化主观评价。
- 每类 landmark 挑选 2 组“检索-检测”结果。
- 共展示 24 组可视化结果。

## 推荐实现顺序

1. 数据检查：确认图片、标注和类别数量。
2. 检索 baseline：先用简单稳定的图像特征跑通全流程。
3. 评价：按文件名前缀计算 P@20、P@40、P@60。
4. 可视化：生成 Top-K 检索拼图。
5. 检测可视化：先用标注框生成可靠展示图。
6. 整理 24 组案例：每类 2 组。
7. 写报告：方法、实验设置、结果分析、可视化案例、改进方向。
8. 录视频：录制 3-5 个测试样例。

## 后续可选增强

- 使用 PaddleOCR、EasyOCR 或 YOLO 系列方法做真实文字检测。
- 比较多种检索方法的 P@K 曲线。
